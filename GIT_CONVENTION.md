# 힐링보이스 프로젝트 커밋 컨벤션

프로젝트의 유지보수와 매일 추가되는 영상 데이터의 내역을 명확히 관리하기 위한 커밋 규칙입니다.

## 📝 메시지 기본 포맷
`유형(범위): 제목`

## 🎬 영상 업데이트 관련 (추천)
매일 추가되는 영상 데이터(`videoData.ts`) 수정 시에는 아래 규칙을 사용합니다.

| 유형 | 의미 | 커밋 메시지 예시 |
| :--- | :--- | :--- |
| **영상 추가** | 새로운 영상 데이터를 추가할 때 | `content(video): add 조혜련 video` |
| **영상 수정** | 기존 영상의 ID, 제목, 일시 등을 수정할 때 | `fix(video): update 김재원 title` |
| **태그/라벨 수정** | 영상의 카테고리나 라벨만 바꿀 때 | `fix(video): change PR label to Teaser` |
| **데이터 구조 변경** | `videoData.ts`의 스키마를 변경할 때 | `refactor(video): add thumbnail URL field` |

## 🛠 기타 일반 작업
| 유형 | 의미 |
| :--- | :--- |
| **feat** | 새로운 기능 추가 (UI 수정 등) |
| **fix** | 버그 수정 |
| **style** | 코드 포맷팅, 세미콜론 누락 등 (로직 변경 없음) |
| **refactor** | 코드 리팩토링 |
| **chore** | 빌드 업무 수정, 패키지 매니저 설정 등 |

---
**팁**: 커밋 메시지는 **영문** 작성을 기본으로 하되, 영상 제목처럼 영문 변환이 애매한 경우 한글을 혼용하여 `add [한글제목] video` 형태로 작성하는 것이 관리 효율상 좋습니다.
